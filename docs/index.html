Документация Smartford OS

Документация движка SmartFord OS: рендеринг .smapp, API (storage, network, prefs, UI) и выполнение цепочек событий.

description Структура

Файл читается функцией RenderApp, разбивается на секции и преобразуется в UI.

[Metadata] id=com.smartfordos.example.myapp title=Пример приложения permissions=["storage","network"] [Button1] type=button label=Нажми меня onClick=handleButtonClick [Text1] type=text id=text1 content=Привет, SmartFord OS! [Events] handleButtonClick=smartfordos.ui.update("text1","Clicked")

code Компоненты

Button

Кнопка создаётся через дублирование шаблона (filled_button, outlined_button и т.д.).

type=button label="Нажми меня" buttonStyle=filled onClick=handleButtonClick position=middle,bottom margin=0,2 id="btn1"

Text

type=text id="text1" content="Привет!" size=18 color=onSurface position=left,top

Цвет автоматически преобразуется через палитру CurrentPalette.

terminal События и цепочки

События выполняются через ExecuteEventChain и поддерживают цепочки команд через ->.

handleButtonClick= smartfordos.api.storage.create_dir(".test") -> smartfordos.api.storage.write_file(".test/new.txt","Hello!") -> smartfordos.api.storage.file.read(".test/new.txt") -> smartfordos.ui.update("text1","{res}")

{res} - результат текущей команды
{prev_res} - результат предыдущей команды

code API

Storage API

smartfordos.api.storage.create_dir("folder") smartfordos.api.storage.write_file("file.txt","data") smartfordos.api.storage.file.read("file.txt")

Network API

smartfordos.network.fetch("https://example.com") smartfordos.network.fetch.post(url, body, headers)

Требует разрешения: network

Prefs API

smartfordos.prefs.set("key","value","String") smartfordos.prefs.get("key") smartfordos.prefs.delete("key")

UI API

smartfordos.ui.update("text1","New text")

System API

smartfordos.system.wait("3000") smartfordos.system.call("hyperlink","https://google.com")

security Разрешения

  • storage - доступ к файловой системе
  • network - HTTP запросы
